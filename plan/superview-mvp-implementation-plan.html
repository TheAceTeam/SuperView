SuperView MVP

A local-first dashboard that turns Codex rollout JSONL into a project timeline, with replayable side-scrolling drill-downs for individual agent runs.

1. Product Goal

SuperView is a local-first Codex log visualization dashboard for complex engineering work. The MVP turns Codex rollout JSONL files into a project timeline, then lets users drill into a single Codex run as a replayable side-scrolling level.

The memorable outcome is: "wow, I have never seen a coding agent from this angle."

2. MVP Scope

  • Scan real ~/.codex/sessions/**/*.jsonl files.
  • Parse and normalize Codex session events.
  • Store redacted evidence in local SQLite.
  • Build project timelines with Product, Architecture, Code, Agent Runs, Verification, and Risks lanes.
  • Open a run replay view with level-like nodes for prompts, tool calls, patches, failures, and verification.
  • Support Bright and Dark Command Center themes.

Out of scope: cloud upload, accounts, teams, watcher, LLM summaries, Pattern Codex, badges, desktop shell, and codex-tui.log.

3. Architecture

React UI
  -> HTTP API now, desktop IPC later
    -> runtime-node
      -> core services
      -> storage
      -> local filesystem and git

The UI must never import fs, path, child_process, or SQLite modules. All local capabilities go through API now and desktop IPC later.

4. Codex Log Integration

  • Primary source: ~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl.
  • Supporting source: ~/.codex/history.jsonl.
  • Ignored for MVP: ~/.codex/log/codex-tui.log.
  • Store redacted payloads, source path, line number, and SHA-256.
  • Reasoning records become markers only; encrypted contents are not displayed.

6. Timeline and Replay

Timeline lanes are Product, Architecture, Code, Agent Runs, Verification, and Risks. Episodes are deterministic and labeled Auto grouped.

Replay maps user prompts to start gates, file reads to context blocks, tool calls to platforms, patches to power-ups, failed commands to hazards, and verification success to finish flags.

7. Frontend Experience

The first screen is the dashboard: Run Ledger on the left, Project Timeline in the center, Evidence Drawer on the right, and replay detail for selected runs. The visual system follows DESIGN.md: warm off-white workspace, dark frosted shell, orange telemetry, and compact dashboard density.

8. Desktop App Considerations

  • Keep React browser-only.
  • Keep parser, redactor, timeline, and replay logic in pure TypeScript modules.
  • Centralize data directory resolution.
  • Add schema_meta(version) from day one.
  • Make Codex home configurable with SUPERVIEW_CODEX_HOME.
  • Use job-based ingest so desktop windows do not freeze.

9. Test Plan

Use fixture rollout files for parser, normalizer, redactor, timeline, and replay tests. Integration tests cover ingest jobs and APIs. E2E tests cover empty state, scan, timeline, replay controls, evidence drawer, and theme switching.

10. Acceptance Criteria

  • pnpm dev starts the dashboard.
  • Scan reads real Codex rollout JSONL files.
  • Timeline and replay views show redacted Codex evidence.
  • Non-git directories do not break ingest.
  • Desktop migration can reuse core/ and storage/.

11. Assumptions

DESIGN.md is the source of truth. Phase 1 is local-only and deterministic. Raw Codex logs stay on the user's machine. Desktop packaging starts only after parser, timeline, replay, and evidence UX are stable.
